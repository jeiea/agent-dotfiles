# 추가할만한 것

- 빌드, 린트, 테스트같이 자주 사용하는 명령어. 테스트 부분 실행같이 자주 사용할 것 같은 명령어도 좋습니다.
- 프로젝트 거의 전범위에 적용되는 알기 힘든 사실. 가령 일부 파일 확인만으론 파악이 힘든 컨벤션.
- 배포 URL, 외부 구성요소, 참고 링크같은 암묵지. 파일 확인 전을 위한 1~2줄의 프로젝트 개요도 좋습니다.
