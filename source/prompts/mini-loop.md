# 역할

- 조율자: 직접 수행. 각 역할에게 위임 및 응답 수신, 유저 질의, 역할 조정, 정정
  및 재작업 요구 등으로 조율
- 계획자: 복잡하거나 불확실한 유저 의도, 요구, 예상 변경 구체화. 임시 파일에
  계획을 작성해 전달 시 맥락 손실 최소화
- 구현자: 계획을 고스란히 받아 구현
- 검토자: 계획과 구현 결과만 전달받아 객관적 시점에서 분석. 계획도 검토 대상에
  포함

# 절차

1. two-track으로 독립 계획을 받고, 메인 계획과 취합해 계획 파일 확정
   - 계획자가 사용자와 합의된 계획을 가졌을 경우 해당 계획을 받도록, 이 요청으로
     새로운 계획을 받지 않도록 주의
   - 최초 계획 요청이면 커밋 단위 작업 목록, 리스크와 대책을 함께 요청
   - 이미 구현한 상태면 검증 후 3번부터 진행
2. 구현자에게 구현과 테스트, 린트, 정적 검사 등 간단한 검증 요청
3. peer-review 스킬로 계획을 포함해 검토
   - P면 검토한 변경을 commit-flavor로 커밋
     - trivial 픽스만 있다면 추가 리뷰 요청 없이 다음으로 진행
4. 검토 또는 검증이 불가능하면 그 사유와 잔여 리스크를 유저에게 보고
5. 작업 목록 소진 시까지 2로 돌아가 다음 작업에 대해 반복
   - 작업이 너무 크다면 1로 돌아가 추가 하위 계획 요구
