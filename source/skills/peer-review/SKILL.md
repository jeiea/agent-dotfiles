# 검토자 선택

- 현재 세션이 Codex면 `claude`, Claude면 `codex` 스킬 우선
- 다른 모델 호출이 어렵다면 서브에이전트 사용 허용

# 원칙

- 편향 방지를 위해 검토자에게 전달할 문맥 최소화
- 산출물 자체, 사용자 요구, 명시 제약, 검토 기준만 전달
- 메인 세션의 추론, 선호 결론, 의심 지점은 기본 전달 금지
- 검토자 응답은 권고. 메인 세션이 최종 판단

# 절차

1. 산출물과 요구사항 정리
2. 요청별 검토 기준 정리
   - 기본: 요구 충족, 오류·모순·누락, 더 단순·견고한 대안
   - 호출처 추가 기준
   - 특정 관점만 검토 요청 시 기본 기준 제외 가능
3. 각 요청을 독립된 검토자 세션에서 같은 턴에 시작

   ```md
   다음 산출물을 수정하지 말고 독립 검토합니다.

   ## 사용자 요구 및 제약

   {원 요청 요약 또는 원문}

   ## 산출물

   {검토 대상}

   ## 검토 기준

   {기본 기준—제외 요청이면 생략}

   {호출처 추가 기준}

   응답 형식:

   # Verdict: P | NP

   # Must fix now:

   현재 범위에서 확인된 요구 위반·결함

   # Consider:

   필수 수정 아닌 개선·취향 제안. 항목별 근거와 기대 이득 명시

   # Follow-up:

   근거 부족·사용자 판단 필요·범위 밖 항목. 필요한 확인 질문·작업 명시

   `NP`는 즉시 수정해야 할 문제가 있을 때만 사용합니다.
   ```

4. 검토자 출처를 보존해 결과를 중요도별 병합하고 중복 제거
5. 각 지적의 사실성·요구 부합·trade-off를 독립 검증 후 템플릿 기준으로 재판정
   - `Consider`: 근거·기대 이득·trade-off가 타당하고 사용자 요구·명시 선호에
     부합할 때만 반영. 낮은 비용만으로 반영 금지
   - 검증 실패: 기각
6. `Follow-up` 처리
   - 현재 범위에서 확인 가능: 확인 후 재판정
   - 사용자 판단 필요: 질문
   - 범위 밖: 후속 작업으로 보고
7. `Must fix now` 반영 후 관련 지적을 낸 모든 검토자 세션 재개
8. 검토 불가 시 사유와 잔여 리스크 보고

# 종료 기준

- 모든 검토자의 `Verdict: P`
- 또는 남은 지적의 기각·`Follow-up` 판정 근거를 메인 세션이 설명 가능
