# 검토자 선택

- 현재 세션이 Codex면 `claude` 스킬 우선
- 현재 세션이 Claude면 `codex` 스킬 우선
- 다른 모델 호출이 어렵다면 서브에이전트 사용 허용
- 같은 모델 재귀 호출, 같은 스킬 재호출은 피함

# 원칙

- 편향 방지를 위해 검토자에게 전달할 문맥 최소화
- 산출물 자체, 사용자 요구, 명시 제약, 검토 기준만 전달
- 메인 세션의 추론, 선호 결론, 의심 지점은 기본 전달 금지
- 검토자 응답은 권고. 메인 세션이 최종 판단

# 절차

1. 산출물과 요구사항 정리
2. 검토자에게 아래 템플릿으로 요청

   ```md
   다음 산출물을 독립 검토합니다.

   ## 사용자 요구

   {원 요청 요약 또는 원문}

   ## 산출물

   {검토 대상}

   ## 검토 기준

   - 더 간단하거나 견고한 대안
   - 요구 충족 여부
   - 오류, 모순, 누락
   - 실행/운영/보안 리스크

   응답 형식:

   # Verdict: P | NP

   # Must fix now:

   # Fix if cheap:

   # Follow-up:

   `NP`는 즉시 수정해야 할 문제가 있을 때만 사용합니다.
   ```

3. `Must fix now` 반영 후 필요 시 같은 검토자 세션 재개해 재검토
4. `Fix if cheap`은 작고 명확한 것만 반영
5. `Follow-up`은 현재 요청 범위를 넘으면 유저에게 보고만 함
6. 검토 불가 시 사유와 잔여 리스크 보고

# 종료 기준

- `Verdict: P`
- 또는 남은 지적이 의도적 보류/범위 밖/취향 차이임을 메인 세션이 설명 가능
