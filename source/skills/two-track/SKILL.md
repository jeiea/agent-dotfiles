# 목적

메인 세션과 다른 모델이 같은 요청을 독립 처리한 뒤, 더 나은 판단과 산출물을
취합.

# 역할

- 메인 세션: 위임, 독립 작업, 비교, 취합, 최종 산출
- 병렬 작업자: 최소 문맥으로 독립 산출

# 작업자 선택

- 현재 세션이 Codex면 `claude` 스킬 우선
- 현재 세션이 Claude면 `codex` 스킬 우선
- 다른 모델 호출이 어렵다면 서브에이전트 사용 허용
- 같은 모델 재귀 호출, 같은 스킬 재호출은 피함

# 원칙

- 먼저 위임. 메인 세션의 사전 조사는 병렬성과 독립성을 해침
- 작업자에게 사용자 요청, 필수 제약, 입력 파일/경로만 전달
- 메인 세션의 가설, 계획, 선호 접근, 중간 조사 결과는 기본 전달 금지
- 두 결과 중 하나를 그대로 고르기보다 장점과 근거를 취합
- 충돌 시 사용자 요구, 저장소 관례, 검증 결과를 우선
- 병렬 수정이나 충돌 위험이 있으면 작업자용 임시 워크트리 복제

# 절차

1. 사용자 요청에서 독립 수행에 필요한 최소 입력만 추출
2. 필요 시 작업자용 임시 워크트리 준비
3. 즉시 병렬 작업자에게 아래 템플릿으로 위임

   ```md
   다음 요청을 독립 수행합니다.

   ## 사용자 요청

   {원문 또는 최소 요약}

   ## 입력/제약

   {파일, 경로, 명시 제약, 완료 기준}

   ## 요청

   - 필요한 조사, 구현, 검토를 스스로 수행
   - 최종 산출물 또는 적용 가능한 제안 작성
   - 핵심 판단 근거와 검증 결과 포함
   - 메인 세션과 비교할 수 있게 변경 파일/명령/리스크 명시
   ```

4. 메인 세션도 독립적으로 작업 수행
5. 결과 비교
   - 요구 충족
   - 정확성, 단순성, 유지보수성
   - 저장소 관례 적합성
   - 검증 가능성
   - 놓친 리스크
6. 더 나은 부분 취합해 최종 산출

# 보고

- 최종 답변에는 채택한 핵심 차이만 간단히 설명
- 버린 제안은 중요 리스크나 트레이드오프가 있을 때만 언급
- 병렬 작업이 불가능했다면 사유와 대체 검토 범위 명시
